List of Documents

The List of documents form display a form similar to the main form of OpenProdoc, with a tree of folders on the left and a list of contained documents on the rigth. The main difference is the lack of automatic display of folder metadata, being needed to open an update form for reviewing the information. The folders displayed will be folders under the element (Department, Product, version,..) selected when open this form.

There are two groups of functions: functions related to folders and functions related to documents. Except the for Add Folder and Add Document, it's necesary to select a document or a folder for activating those operation.

  • Folders
    • Add Folder: Displays a form for adding a folder under the current selected folder. Although the folder can be of any type, it's better to use it for creating a folder type different from the SMO folder types (Producst, Versions, Issues, Departments and Software Companies), avoiding errors in structure of information.
    • Update Folder: Displays a form for updating the selected folder.
    • Delete Folder: Displays a form for confirmation before deleting the selected folder and ALL its contained folders and documents.
  • Documents
    • Add Document: Displays a form for adding a document of any type under the current selected folder.
    • CheckOut Document: Locks the selected document for exclusive edition.
    • Update Document: Displays a form for updating the selected document(previously checked out by the current user).
    • CheckIn Document: Creates a new version of the selected document (previously checked out by the current user).
    • Cancel CheckOut Document: Cancels the locking and the updates made after the last locking.
    • ListVersions of a Document: Displays a list of the versions of the selected document.
    • Delete Document: Displays a form for confirmation before deleting the selected document and ALL its versions.

In SMO Tree there is a complete view of operations and forms.
