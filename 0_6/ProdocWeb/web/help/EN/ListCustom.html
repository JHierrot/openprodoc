Customizations List

By selecting the option to manage Customizations, will appear the form "list of Customizations". This form allows you to maintain all the system Customizations.

The customization allows you to configure the appearance of OPD clients, language and elements such as date formats. This allows OPD to integrate with applications or to adapt to an organization or accessibility needs.

Users are assigned a customization, so that multiple users can simultaneously access the same system OPD, each in their own language and personal appearance.

To filter the Customization or list of Customizations you want to review, just enter part of the name in the text box and press "ok". The list of Customizations that meet the conditions will be shown on the results table. Pressing the button without entering any value you see all the elements on which the user has permission. The results table shows the data:

  • Identification name
  • Description of the Customization
  • Language for the user interface
  • Input and display format of the Dates
  • Input and display format of the Date-time
  • Path parameter for the Web client, which contains all the elements (CSS, images, etc.)
  • Fonts to be used on the Java Swing client
  • Name of last user who modified the Customization
  • Date and time of last modification

This results table can be sorted by selecting the header of each column. You can also change the size of each column by dragging the separator line in the headers.

Above the table of results there are several buttons for performing operations on the selected item. The available operations are:

  • Add a new element
  • Delete the selected element (if it is not used in some document or folder)
  • Modify the selected element
  • Create a new item as a copy of the selected item
  • Export the selected item
  • Export all items listed
  • Import from file one or more previously saved items

It should be noted on export and import, that some elements may have dependency on others, so you must export all related and imported at the time of it in the proper order.

In the event of an error (lack of user permissions, data inconsistency, etc.), the operation is canceled and will present the reason for the error to the user.
