User Interface OpenProdoc

The User Interface of OPD is build following the next criteria:

  • Having a simple and intuitive interface, which minimizes the need for manuals.
  • Having an interface as similar as possible for both clients (Web and Swing).
  • Take advantage as far as possible the characteristics of each environment.
  • Be parameterized to allow it to fit the needs of each organization or individual.

As in general, the requirements are somewhat contradictory, so has been sought a compromise. As a result, OPD clients have the following characteristics and behavior:

  • The behavior of buttons and menu options is always on the selected item of any type whatsoever, in query or administration functions.
  • Given the advantages to using tables that can be ordered and adjusted, so has been defined on the Swing client (to be included in future versions of Web Client).
  • It has also been taken advantage of the use of pop-ups on Swing, but not in the Web client, for the problems of accessibility and behavior "nonstandard."
  • No administration functions has been included in this first version of the Web client but will be included in future.
  • For Swing client, more administration oriented, the only permitted settings are the change in character fonts.
  • In the Web client you can change the CSS style sheets, images and even javascript, allowing a nearly complete change in appearance.
  • Given on the one hand the similarity of Web forms and Swing and on the other the potential difference there can be due to changes in CSS, it was decided to include Swing forms mainly in the documentation, since will change less.
  • To facilitate the adaptation and understand the effect of each element, is included in the Web Client a series of example settings. To create a new one, just copy the appropriate folders of one example (contained in css, img and js) and create another style. The fact that JavaScript is differentallows even modify or create items so that would not be possible only with css.
