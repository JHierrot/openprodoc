OpenProdoc is an ECM Document Management system developed in Java as Open Source

This project is maintained by JHierrot

OpenProdoc is an ECM Document Management system with the characteristics:

What is new in version 1.0:

  • Task Support
    • Scheduled of event triggered.
    • Delete and purge of old documents/cases
    • News and purge Reports
    • Scheduled Import/Export
    • Automatic update and conversion
  • Extended Architecture for easier administration and more security.
  • Added support for storage in Amazon AWS S3 (and API compatible systems as Atmos and OpenStack)
  • Added interface in Catalan language.

What is new in version 0.8:

  • Thesaurus Support
    • Complete Thesaurus management.
    • Skos-rdf standard support for import-export.
    • Multiple thesaurus.
    • Validation of selected metadata against thesaurus.
  • Modification of definitions already in use.
  • Import of document scanned and classified in Kofax Capture or Abbyy Flexicapture

OpenProdoc maintains its characteristics:

  • A complete portable version (Linux, Windows, Mac)
  • Multi-platform (Java)
  • Multi-database (Derby, MySQL, Oracle, DB2, PostgreSQL, SQLServer, SQLLite, HSQLDB)
  • Low requirements for the engine (can work without a J2EE server)
  • Several ways for Authentication (Ldap, DDBB, OS, Own system)
  • Different ways to store documents (FileSystem, BLOB,ftp, Reference)
  • Object oriented definitions for documents and folders (including inheritance)
  • Fine granularity of administration and permissions, allowing delegation of different functions.
  • Multi-language (English, Spanish and portuguese)
  • Thin (Web) and Thick (Swing) Clients
  • Free and OpenSource

OpenProdoc es un Gestor Documental con las características:

Las principales novedades de la versión 1.0 son:

  • Soporte de tareas
    • Programadas o por eventos
    • Borrado y purgado de expedientes antiguos
    • Informes de expurgo y boletines de novedades
    • Importaciones y exportaciones programadas
    • Actualización y conversión automatica
  • Arquitectura extendida para facilitar la administración y aumentar seguridad.
  • Añadido soporte para almacenamiento en Amazon AWS S3 (y sistemas con un API compatible como Atmos y OpenStack)
  • Añadido interfaz en catalán

Las principales novedades de versión 0.8 son:

  • Soporte de Tesauros
    • Gestión completa de múltiples Tesauros.
    • Importación y Exportación en formato estándar SKOS.
    • Validación de metadatos de tipos documentales y carpetas contra valores de los tesauros.
  • Modificación de Tipos de documento y carpeta ya en uso, con documentos introducidos.
  • Importación de documentos digitalizados y clasificados desde Kofax Capture y Abbyy Flexicapture

Además, mantiene el resto de características:

  • Dispone de una vertión totalmente portable (Linux, Windows, Mac)
  • Multi-platforma (Java)
  • Multi-base de datos (Derby, MySQL, Oracle, DB2, PostgreSQL, SQLServer, SQLLite, HSQLDB)
  • Bajos requerimientos para el motor (puede utilizarse sin servidor J2EE)
  • Diversas formas de autenticación para cada usuario (Ldap, DDBB, OS, propietario)
  • Diferentes formas de almacenar documentos (Sistema de archivos, BLOB,ftp, Referencia web)
  • Definiciones de carpetas y documentos flexibles y orientadas a objetos (incluyendo herencia)
  • Granularidad muy fina de administración y permisos, permitiendo delegar diferentes funciones.
  • Multi-Idioma (Castellano, Inglés y Portugués sin manual)
  • Clientes pesado y ligero (web)
